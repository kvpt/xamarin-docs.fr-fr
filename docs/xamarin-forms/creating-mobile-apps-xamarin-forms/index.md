# <a name="creating-mobile-apps-with-xamarinforms-book"></a>Création d’applications mobiles avec Xamarin.Forms livre

[![Télécharger](~/media/shared/download.png) l’échantillon Télécharger l’échantillon](https://github.com/xamarin/xamarin-forms-book-samples)

<img src="images/cover-sml.png" title="Création d’applications mobiles avec Xamarin.Forms" align="left" />Le livre *Creating Mobile Apps with Xamarin.Forms* de Charles Petzold est un guide pour apprendre à écrire des applications Xamarin.Forms. La seule condition préalable est la connaissance du langage de programmation de C. Le livre fournit une exploration approfondie dans l’interface utilisateur Xamarin.Forms et couvre également l’animation, MVVM, déclencheurs, comportements, mises en page personnalisées, rendus personnalisés, et bien plus encore.

Le livre a été publié au printemps 2016 et n’a pas été mis à jour depuis. Il ya beaucoup dans le livre qui reste précieux, mais certains des [matériaux est obsolète](#ways-in-which-the-book-is-outdated), et certains sujets ne sont plus entièrement corrects ou complets.

## <a name="ways-in-which-the-book-is-outdated"></a>Façons dont le livre est dépassé

Depuis la publication de *Creating Mobile Apps avec Xamarin.Forms*, plusieurs nouvelles fonctionnalités ont été ajoutées à Xamarin.Forms. Ces nouvelles fonctionnalités sont décrites dans des articles individuels de la documentation [Xamarin.Forms.](/xamarin/)

D’autres changements ont fait en sorte qu’une partie du contenu du livre est désuète :

### <a name="net-standard-20-libraries-have-replaced-portable-class-libraries"></a>.NET Standard 2.0 bibliothèques ont remplacé les bibliothèques de classe portable

Une application Xamarin.Forms utilise généralement une bibliothèque pour partager du code entre les différentes plates-formes. À l’origine, il s’agissait d’une bibliothèque de classe portable (PCL). Il ya beaucoup de références aux PCL tout au long du livre et les résumés chapitre.

La bibliothèque de classe portable a été remplacée par une bibliothèque .NET Standard 2.0, telle que décrite dans l’article [.NET Standard 2.0 Support in Xamarin.Forms](~/xamarin-forms/internals/net-standard.md). Tous les échantillons de [code](https://github.com/xamarin/xamarin-forms-book-samples) du livre ont été mis à jour pour utiliser les bibliothèques .NET Standard 2.0.

La plupart des informations contenues dans le livre concernant le rôle de la Bibliothèque de classe portable demeurent les mêmes pour une bibliothèque .NET Standard 2.0. Une différence est que seul un PCL a un «profil» numérique. En outre, il ya quelques avantages à .NET Standard 2.0 bibliothèques. Par exemple, chapter 20, [Async et File I/O](summaries/chapter20.md) décrivent comment utiliser les plates-formes sous-jacentes pour effectuer le fichier I/O. Cela n'est plus nécessaire. La bibliothèque .NET Standard 2.0 prend en charge les classes [System.IO](xref:System.IO) familières pour toutes les plateformes Xamarin.Forms.

La bibliothèque .NET Standard 2.0 permet également aux [`HttpClient`](xref:System.Net.Http.HttpClient) applications Xamarin.Forms [`WebRequest`](xref:System.Net.WebRequest) d’utiliser pour accéder aux fichiers sur Internet plutôt que sur d’autres classes.

### <a name="the-role-of-xaml-has-been-elevated"></a>Le rôle de XAML a été élevé

*Créer des applications mobiles avec Xamarin.Forms* commence par décrire comment écrire des applications Xamarin.Forms à l’aide de C. L’Extensible Application Markup Language (XAML) n’est pas introduit avant [le chapitre 7. XAML vs Code](summaries/chapter07.md).

XAML a maintenant un rôle beaucoup plus important dans Xamarin.Forms. Les modèles de solutions Xamarin.Forms distribués avec Visual Studio créent des fichiers de pages XAML. Un développeur utilisant Xamarin.Forms devrait se familiariser avec XAML dès que possible. La section [eXtensible De la langue de markup d’application (XAML)](~/xamarin-forms/xaml/index.yml) de la documentation Xamarin.Forms contient plusieurs articles sur XAML pour vous faire démarrer.

### <a name="supported-platforms"></a>Plateformes prises en charge

Xamarin.Forms ne prend plus en charge Windows 8.1 et Windows Phone 8.1.

Le livre fait parfois référence à la _Windows Runtime_. Il s’agit d’un terme qui englobe l’API Windows utilisé dans plusieurs versions de Windows et Windows Phone. Les versions plus récentes de Xamarin.Forms se limitent à prendre en charge la plate-forme Windows universelle, qui est l’API pour Windows 10 et Windows 10 Mobile.

Une bibliothèque .NET Standard 2.0 ne prend en charge aucune version de Windows 10 Mobile. Par conséquent, une application Xamarin.Forms à l’aide d’une bibliothèque .NET Standard ne s’exécutera pas sur un appareil Windows 10 Mobile. Les applications Xamarin.Forms continuent de s’exécuter sur le bureau Windows 10, versions 10.0.16299.0 et plus.

Xamarin.Forms a un aperçu des plates-formes [Mac,](~/xamarin-forms/platform/other/mac.md) [WPF](~/xamarin-forms/platform/other/wpf.md), [GTK](~/xamarin-forms/platform/other/gtk.md)et [Tizen.](~/xamarin-forms/platform/other/tizen.md)
